# iCloush 智慧工厂小程序 — V9 深度重构更新日志

**代号**: 审查闭环 + 负责人可视化

## 重构概览

V9 版本完成了四项深度重构，涵盖真机兼容性修复、全局数据联动、任务审查业务闭环、以及任务列表负责人信息补全。

---

## Phase 1: 2.5D 沙盘真机可见性修复

- **问题**: WebP 格式底图在部分真机上不可见
- **方案**: 将 `factory_f1.webp` / `factory_f2.webp` 转换为 JPG 格式
- **文件**: `images/factory_f1.jpg`, `images/factory_f2.jpg`
- **影响页面**: `pages/index/index`（总览页沙盘）

## Phase 2: 沙盘工区定位修复

- **问题**: 工区热区定位与 2.5D 等轴测底图不匹配
- **方案**: 重新校准所有 13 个工区的 `pos` 坐标（left/top/width/height）
- **文件**: `utils/mockData.js` — ZONES 数据

## Phase 3: 排班数据全局联动

- **问题**: 总览页与排班管理页的排班数据各自独立，修改不同步
- **方案**: 通过 `app.globalData.scheduleData` 实现跨页面数据共享
- **机制**: `onShow` 生命周期刷新 + 内存状态驻留
- **影响页面**: `pages/index/index`, `pages/schedule/index`

## Phase 4: 任务审查业务闭环

### mockData.js 变更
- `/task/submit` 路由: 提交后状态变为 **3（待审核）**，不再直接完成
- 新增 `/task/review` 路由: 支持 `pass`（通过→状态4）和 `fail`（驳回→状态2 + is_rejected 标记）

### task-detail 页面变更
- 新增 **驳回警告横幅**: 红色背景 + 驳回原因展示
- 新增 **审核状态展示区**: 待审核/已通过/已驳回三种状态卡片
- 新增 **管理审核工作台**: 仅管理员可见，包含「审核通过」和「驳回」按钮
- 新增 **负责人信息**: 头像圆圈 + 姓名，从 USERS 数据匹配
- 驳回后任务退回「进行中」状态，提交按钮变为「重新提交 → 待审核」

### task-list 页面变更
- 驳回任务显示 **红色边框** 高亮
- 驳回任务显示 **「已驳回·需重新提交」** 标记

## Phase 5: 任务列表负责人信息

- 任务卡片底部新增 **负责人头像圆圈 + 姓名**
- 通过 `/api/v1/users` 接口获取用户列表，与任务的 `assigned_to` 字段匹配
- 未分配任务显示灰色 `?` 占位符 + 「未分配」文字
- 用户列表缓存至 `app.globalData._cachedUsers` 供其他页面复用
- `onShow` 生命周期自动刷新（从详情页返回时同步审核状态变更）

## 任务状态机

```
待接单(0) → 已接单(1) → 进行中(2) → 待审核(3) → 已完成(4)
                                  ↑                    ↓
                                  └── 驳回(is_rejected) ←┘
```

- **状态 0**: 待接单（灰色）
- **状态 1**: 已接单（蓝色）
- **状态 2**: 进行中（蓝色）
- **状态 3**: 待审核（黄色）
- **状态 4**: 已完成（绿色）
- **is_rejected**: 驳回标记（红色边框 + 红色标签）

## 测试要点

1. **任务提交 → 审核流程**: 员工登录 → 进入任务详情 → 拍照取证 → 提交 → 状态变为「待审核」
2. **管理员审核**: 管理员登录 → 任务列表筛选「待审核」→ 进入详情 → 点击「审核通过」或「驳回」
3. **驳回重新提交**: 驳回后任务退回「进行中」→ 任务列表红框高亮 → 详情页显示驳回横幅 → 可重新提交
4. **负责人显示**: 任务列表每张卡片底部显示负责人头像和姓名
5. **沙盘可见性**: 总览页 2.5D 工厂沙盘在真机上正常显示
6. **排班联动**: 总览页修改排班 → 切换到排班管理页 → 数据同步
