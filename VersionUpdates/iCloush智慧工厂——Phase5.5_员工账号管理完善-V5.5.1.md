## 版本概述

本次版本针对员工管理模块的三个核心问题进行了完善：停用账号无视觉反馈、缺少删除账号能力、编辑时无法修改账号密码。重构后，员工账号的完整生命周期（创建 → 编辑 → 停用 → 恢复 → 删除）均有对应的 UI 交互和 API 支持。

---

## 功能变更

### 1. 停用账号视觉反馈

停用账号后，员工卡片立即呈现明确的视觉差异，无需猜测是否已停用。

| 元素 | 正常状态 | 停用状态 |
|------|----------|----------|
| 卡片整体 | 正常不透明度 | opacity: 0.45，背景变暗 |
| 头像渐变 | 原色渐变 | 灰色渐变 (#555 → #333) |
| 姓名/积分 | 白色/金色 | 灰色 (#555) |
| 技能标签 | 深色背景 | 更暗背景 + 灰色文字 |
| 状态指示灯 | 绿色 (active) | 灰色 (inactive) |
| 标签 | 多岗标签 | 红色「已停用」标签 |

弹窗内同样显示：头像灰化 + 大号「已停用」标签。

### 2. 恢复账号

停用状态下，弹窗底部的「停用账号」按钮自动替换为绿色的「恢复账号」按钮，确认后调用 `/api/v1/users/:id/restore` 恢复为 active 状态。

### 3. 永久删除账号

弹窗底部新增「永久删除账号」按钮（虚线边框，低调设计），采用二次确认机制：

- 第一次确认：说明删除范围（账号和密码），并告知历史数据（发票、票据等）不会被删除
- 第二次确认：最终确认删除

删除后从 USERS 和 WHITELIST（登录白名单）中同时移除。

### 4. 编辑时显示账号密码

进入编辑模式后，新增「登录账号」和「密码」两个编辑字段：

- **登录账号**：显示当前账号，可直接修改
- **密码**：默认为空，留空表示不修改；输入新密码则重置（至少6位）
- 密码字段支持显示/隐藏切换

查看模式下也显示登录账号（只读）。

### 5. Mock 数据增强

| 变更 | 说明 |
|------|------|
| USERS 数据 | 每条记录新增 `username` 字段 |
| u006 刘芳 | status 改为 `inactive`（停用示例） |
| PUT /users/:id | 同步更新 USERS + WHITELIST |
| POST /users/:id/disable | 更新 USERS 中的 status 为 inactive |
| POST /users/:id/restore | 新增路由，恢复 status 为 active |
| DELETE /users/:id | 新增路由，从 USERS 和 WHITELIST 中移除 |
